Day 2: Variables Don't Store Values (Python Memory Model)

April 23, 2026 · 1 min · Nitin S Kulkarni

If you think x = 10 means “x stores 10”…
you are already misunderstanding Python.

Today, we fix that.

Today’s Goal#

By the end of today, you will:

  • Understand how Python stores data in memory
  • Learn references vs values
  • Understand mutability
  • Avoid hidden bugs

The Illusion#

You write:

x = 10
y = x

You think:

“x and y both store 10”

Wrong.

What Actually Happens#

Memory: [10] ← object

x ─────┐

    ├──→ [10]

y ─────┘

Variables store references, not values.

Key Rule#

Everything in Python is an object.

Identity vs Value#

x = 10
y = 10

print(x == y)
print(x is y)

Mutability#

Immutable:

x = 10
y = x
x = x + 5

Mutable:

a = [1,2,3]
b = a
a.append(4)

Function Behavior#

def add_item(lst):
    lst.append(100)

my_list = [1,2,3]
add_item(my_list)

Mutable Default Trap#

def add_item(item, lst=[]):
    lst.append(item)
    return lst

Fix:

def add_item(item, lst=None):
    if lst is None:
        lst = []
    lst.append(item)
    return lst

Reference Counting#

Object deleted when ref count = 0.

Task#

Use id(), is, and experiment with mutability.

Final Insight#

Variables are labels pointing to objects.

See you in Day 3.
