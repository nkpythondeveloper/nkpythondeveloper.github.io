Day 3: Your Data Structures Are Not What You Think (Lists, Tuples, Sets Internals)

April 23, 2026 · 1 min · Nitin S Kulkarni

You think a list is just “a collection of items.”
It’s not.

It’s a memory-managed data structure with trade-offs.

Today, you stop using data structures blindly.

Today’s Goal#

By the end of today, you will:

  • Understand how lists, tuples, and sets work internally
  • Learn their time and space complexity
  • Choose the right structure based on behavior and cost

The Illusion#

You write:

data = [1, 2, 3]

Reality:

This is a dynamic array of references

Lists — Dynamic Arrays#

  • Stores references
  • Resizes dynamically

Resizing#

Append mostly O(1), occasionally expensive

Complexity#

  • Access → O(1)
  • Insert → O(n)
  • Search → O(n)

Tuple#

  • Immutable
  • Memory efficient

Set#

  • Hash table
  • O(1) lookup

Example#

lst = [1,2,2,3]
list(set(lst))

Insight#

Choose structure based on cost

Task#

Compare list vs set performance
